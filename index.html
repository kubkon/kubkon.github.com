Hi! My name is Jakub, but you might know me by my Github handle kubkon. I'm a compiler engineer, and an open-source addict.

I am a Zig core team member, and a paid contractor to Zig Software Foundation, where I mainly work on the compiler (backends and linkers) with occassional fixes to anything macOS-related. I am also one of the authors of the emerald linker which serves as a drop-in replacement for system linkers on macOS, Linux, Windows and WASM (disclaimer: it is still under development but feel free to try it out!).

Previously, I used to contribute to the Rust/Wasm/WASI ecosystem but due to time constraints, I had to dial this down somewhat (if only days were longer...). However, that doesn't mean you will never see me contribute to this fanastic ecosystem ever again - stay tuned for more updates!

In this blog, I'll try to reflect on the more interesting aspects of my journey through the world of programming. If you would like to discuss any of the posts, feel free to drop me a line at moc.aknokbukaj@nokbuk.

My most notable contributions in no particular order

  • Zig, where I am one of the core team members and I'm actively working on the backends and linkers for the self-hosted compiler.
  • emerald, a system linker drop-in replacement. This project came about as an answer to all the problems we've been having with LLD in Zig when linking MachO binaries and dynamic libraries. Since then it matured into a full-fledged system linker replacement project like the mold linker.
  • zig-yaml, a YAML parser library written entirely in Zig. I hear that folks are actually using it which always makes me smile and chuckle a little since the project came about when I was first writing a parser for Apple's text-based definition (.tbd) format for dynamic libraries.
  • wasmtime, a Wasm and WASI runtime curated by the Bytecode Alliance community, where I contributed and maintained the Rust implementation of the WASI spec, and also contributed a little to the fantastic cap-std library - a capability-based approach at Rust's libstd.
  • WASI, or WebAssembly System Interface, where I worked on the spec as well as the tooling such as witx.
