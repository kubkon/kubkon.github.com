Hi! My name is Jakub Konka, but you might know me by my Github handle kubkon. I'm an open-source addict and recently honoured to have become Zig core team member, currently focusing on Zig's first-class macOS support. I am the main author and contributor to the self-hosted Mach-O linker in Zig, and I have been working hard ensuring that Zig's stage1 compiler and libstd are usable on the latest macOS Big Sur running on Apple Silicon (ARM64) - no kidding, you will be able to code in Zig natively on ARM64 macOS with Zig 0.7.0 release!

Previously, I used to contribute a lot to the Wasm/WASI ecosystem but due to time constraints, I had to dial this down somewhat (if only days were longer...). However, that doesn't mean you will never see me contribute to this fanastic ecosystem ever again - stay tuned for more updates!

Since all of this work is done in my spare time on a voluntary basis, if you like my work, please sponsor me on GitHub.

In this blog, I'll try to reflect on the more interesting aspects of my journey through the world of programming. If you would like to discuss any of the posts, feel free to drop me a line at moc.aknokbukaj@nokbuk or message me on Twitter at @kubkon.

My most notable contributions

  • Zig, where I'm actively working on the self-hosted Mach-O linker and enhancing Zig's macOS support, but not only! I'm also contributing to the self-hosted codegen stage, most recently adding basic support for ARM64 to the Zig's self-hosted compiler. Last but not least, I've also contributed Windows NT syscalls wrappers to Zig's libstd, and curated Zig's support for WASI. BTW, in case you missed this, Zig offers now tier 1 support for WASI!
  • wasmtime-zig, a Zig embedding of wasmtime, my toy project that allows you run Wasm/WASI from within your Zig code by embedding the wasmtime runtime. I've not had the time to give this one proper love it deserves, so it's pretty dated by now, but I do intend to get back to it when I get the chance (come on days, become longer!).
  • wasmtime, a Wasm and WASI runtime curated by the Bytecode Alliance community, where I contributed and maintained the Rust implementation of the WASI spec, and also contributed a little to the fantastic cap-std library - a capability-based approach at Rust's libstd.
  • WASI, or WebAssembly System Interface, where I worked on the spec as well as the tooling such as witx.
